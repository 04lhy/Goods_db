# 数据库系统小学期第三周周计划 — 王宇航（并发控制 + 事务管理）

1. 完善 Lock Manager 以支持网络层多线程场景——扩展表级意向锁（IS/IX/S/SIX/X）用于 DDL 与 DML 的互斥控制，实现锁等待超时机制（lock_wait_timeout 可配置，默认 30 秒，超时自动回滚，作为 waits-for graph 死锁检测的补充），暴露统计接口（GetLockCount / GetDeadlockCount / GetAvgWaitTime）供管理面板展示。

2. 完善 Transaction Manager——支持 SAVEPOINT 保存点（事务内部分回滚到指定点）和 RELEASE SAVEPOINT，实现 XID 分配器（单调递增 64 位事务 ID + 持久化到 header page，服务器重启后继续递增），为 Binary Log 提供事务 ID。

3. 将 Lock Manager 集成到 ConnectionHandler 执行流程——每个连接维护活跃事务状态，SELECT 根据隔离级别决定加锁策略（READ COMMITTED 不加 S 锁 / REPEATABLE READ 对扫描行加 S 锁），INSERT/UPDATE/DELETE 对目标行加 X 锁 + 表级 IX 意向锁，DDL（CREATE/DROP TABLE）对表加 X 锁阻止并发 DML。

4. 实现事务 SQL 命令——START TRANSACTION / BEGIN / COMMIT / ROLLBACK / SET TRANSACTION ISOLATION LEVEL 的 Parser → Binder → Executor 全链路，支持 autocommit 模式（每条语句自动提交，默认开启）和显式事务模式。

5. 进行并发场景测试——多客户端同时 UPDATE 同一行验证锁排队和死锁检测，混合读写场景（80% SELECT + 15% UPDATE + 5% INSERT）验证隔离级别正确性，热点行竞争场景验证锁等待超时回滚机制。

6. 输出 Lock Manager + Transaction Manager 完整设计文档（2PL 协议状态机 + 死锁检测算法 + 四种隔离级别对比 + 并发测试报告）。
