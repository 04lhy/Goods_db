# 数据库系统小学期第三周周计划 — 肖清峰（网络协议层）

1. 设计 goods_db 网络协议包格式规范——Packet Header（3 字节 payload_length + 1 字节 sequence_id）、Command Packet（command_byte + sql_string）、OK/ERR/ResultSet Response Packet 的字节级布局定义，输出协议设计文档 docs/design/network_protocol.md。

2. 实现 Protocol 抽象基类——store 方法族（store_null / store_integer / store_float / store_string）、send 方法族（send_result_metadata / send_row / send_ok / send_error / send_eof），定义完整虚函数表，作为 Protocol_text 和 Protocol_binary 的统一接口。

3. 实现 Protocol_text 文本协议——所有数据按文本行格式序列化（列值 \t 分隔，行 \n 结束），Columns: N → ColDef: name type len → #ROWS → Row: val1\tval2... → EOF 的完整 wire format，支持 telnet/nc 手动调试。实现 Protocol_binary 二进制协议作为高性能备选。

4. 实现 Connection 类——封装 socket 读写：ReadPacket（阻塞读取 4 字节 header + payload → 最大 16MB 限制）、WritePacket（按协议格式封装 header + payload 发送）、WritePacketStr（便捷字符串发送）、Close，管理连接生命周期状态机（INIT → AUTH → READY → QUERYING → CLOSING）。

5. 实现 SocketServer——CreateSocket (bind + listen) / Accept (阻塞等待连接返回 client_fd) / Close，支持可配置 host 和 port，backlog 默认 128。

6. 与刘晗阳联调 Web 前端协议对接——确保 GoodsDbProtocol（Web 服务端的 Qt-free TCP 客户端）通过 Protocol_text 与服务器正确通信：认证 → QUERY → 结果解析 → 断开全链路。输出网络协议层完整设计文档（类层次图 + 连接状态机 + 协议包字节级规范）。
