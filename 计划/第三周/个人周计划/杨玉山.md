# 数据库系统小学期第三周周计划 — 杨玉山（表与元组层 / 安全管理子系统）

1. 设计 goods_db 安全模型——goods_db.user 用户账户表（host/user/password_hash/salt，SHA-256 密码哈希）、goods_db.db 数据库级权限表（host/user/db_name + 8 种操作权限 ENUM）、goods_db.tables_priv 表级权限表（host/user/db_name/table_name + 列级权限 SET），三张 ACL 系统表以普通 TableHeap 存储但标记为系统表（不可被普通 DROP TABLE 删除）。
2. 实现用户认证核心函数——check_connection(host, user, password) → bool：查 goods_db.user 验证 host+user → SHA-256(password+salt) 比对 password_hash → 返回认证结果；check_access(user, host, db, table, operation) → bool：按 host→user 优先级匹配权限行 → 检查 operation 对应的权限字段。
3. 实现 GRANT / REVOKE SQL 命令——扩展 Parser → Binder → Planner → Executor 全链路：GRANT SELECT,INSERT ON db.table TO 'user'@'host' → 插入/更新 goods_db.tables_priv 权限行；REVOKE INSERT ON db.table FROM 'user'@'host' → 更新权限字段为 'N'。
4. 实现用户管理 SQL 命令——CREATE USER / DROP USER / ALTER USER / SET PASSWORD 的 Parser → Binder → Executor 全链路，以及 FLUSH PRIVILEGES（重新从系统表加载权限缓存到内存 HashMap 提升检查性能）。
5. 将 check_connection() 接入肖清峰的 ConnectionHandler 认证流程——客户端连接后服务端发送 handshake 包（server_version + random challenge），客户端返回用户名+密码 hash，服务端调用 check_connection 验证。
6. 实现安全审计跟踪（认证失败记录 Error Log + IP 连续失败 N 次临时封锁）和 SHOW GRANTS 命令（查看指定用户的全部权限）。输出安全管理子系统完整设计文档（ACL ER 图 + 认证流程图 + 权限检查矩阵 + 密码安全方案）。
