# 数据库系统小学期第三周周计划 — 唐瑞涛（线程管理 + 启动流程 + CI/CD）

1. 实现 ThreadPool 线程池——预创建 N 个 worker 线程（N 可配置，默认 = CPU 核心数），线程安全任务队列（基于 std::function + mutex + condition_variable），Submit(task) 提交任务并返回 future<Result>，支持优雅关闭 Shutdown()（等待所有任务完成）和强制关闭 ShutdownNow()（丢弃未执行任务）。

2. 实现 goods_db 服务器启动流程 main() in goods_db_main.cpp——解析命令行参数（--host/--port/--threads/--datadir）→ 初始化 DiskManager + BufferPoolManager + Catalog → 注册存储引擎（register_engine）→ 初始化 AuthManager → 初始化 LogManager → 创建 ExecutionEngine → 启动 ThreadPool → SocketServer Listen → accept 循环，每连接 Submit 到 ThreadPool。

3. 实现 ConnectionHandler 连接处理器——管理单个客户端连接完整生命周期：ReadPacket 接收数据 → 握手（发送 greeting）→ 认证（解析 AUTH 包 → AuthManager 验证）→ 命令分发（PING/QUIT/QUERY）→ 执行 SQL（调用 ExecutionEngine::ExecuteSQL）→ Protocol_text 序列化结果集 → WritePacket 发送 → 断开清理。维护连接状态机（INIT → AUTH → READY → QUERYING → CLOSING）。

4. 实现服务器优雅关闭——SIGTERM/SIGINT 信号处理：停止 accept 新连接 → 等待所有活跃连接完成 → FlushAllPages 落盘脏页 → Catalog Save → 关闭所有资源 → 退出。编写完整的启动/关闭生命周期测试。

5. 完成全部服务集成——ThreadPool + ConnectionHandler + Protocol + Auth + Log + ExecutionEngine 在 main() 中完整串联，服务器可启动监听 → 接受多客户端连接 → 认证 → 执行 SQL → 返回结果 → 记录日志的全链路运行。

6. 更新 CI 流水线——增加网络层/日志层编译目标、多客户端并发连接集成测试，确保 CI 全绿（编译 + 全量单测 + 集成测试）。输出服务器部署文档和配置参数清单。
