# 数据库系统小学期第三周周计划 — 李锦华（日志子系统 + Web 前端辅助）

1. 设计日志系统三层架构——ErrorLog（服务器错误/警告/信息，格式 [timestamp][level][thread_id] message，可配置级别过滤：DEBUG/INFO/WARN/ERROR/FATAL，双输出 stderr + 文件）、QueryLog（所有 SQL 查询记录，格式 [timestamp][thread_id][user@host][db][exec_time] sql_text）、BinaryLog（WAL 消费端，QUERY/ROW_INSERT/ROW_UPDATE/ROW_DELETE/XID 五种事件类型，事务提交前写入 + fsync 刷盘，文件轮转 1GB 切割），统一由 LogManager 管理。

2. 实现 ErrorLog 和 QueryLog——ErrorLog::Log(level, file, line, format, ...) → 格式化 + 时间戳 + 线程 ID → 写入文件 + stderr；QueryLog::LogQuery(user, host, db, sql, exec_time, rows) → 按格式写入查询日志文件。实现日志管理 SQL 命令（SHOW LOGS / SHOW BINLOG EVENTS / PURGE BINARY LOGS / FLUSH LOGS）。

3. 实现 BinaryLog 核心——BinlogWriter 类：WriteEvent(event_type, txn_id, payload) → 构建事件包 → 写入当前 binlog 文件 → 超 max_binlog_size 时 Rotate 切换到序号文件。实现 binlog 索引文件（goods_db_binlog.index）记录所有 binlog 文件名和起始位点。

4. 实现 goods_db_binlog 命令行工具——goods_db_binlog [options] binlog_file，支持 --verbose 输出事件内容，参照 mysqlbinlog 格式：QUERY 事件输出原始 SQL，ROW 事件输出 ### INSERT/UPDATE/DELETE 行数据。

5. 将日志系统接入全服务器——ErrorLog 自动记录启动/关闭/错误/异常，QueryLog 在 ConnectionHandler 每条 SQL 执行时自动记录，BinaryLog 在 DML 操作的事务提交前写入 binlog 事件。

6. 协助刘晗阳进行 Web 前端网络对接——确保 GoodsDbProtocol 通过 Protocol_text 与服务器正确通信，协助排查协议解析和连接管理问题。输出日志子系统完整设计文档（架构图 + 事件格式规范 + 管理命令参考）。
