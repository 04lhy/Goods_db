# 数据库系统小学期第二周周计划 — 李锦华（磁盘与缓冲层 + 桌面客户端辅助）

1. 实现 SortExecutor——Init 阶段收集全部 tuple 按 ORDER BY 键排序（std::sort 内存排序），Next 阶段依次 emit 排序后 tuple。支持单列/多列排序和 ASC/DESC 混合。
2. 实现 ExternalMergeSortExecutor——内存不足时自动降级为外部归并排序：数据分批读入内存排序生成 sorted run 写入临时页 → 多路归并（最小堆选择最小 tuple emit），确保与 BufferPoolManager 的页管理正确对接。
3. 实现 AggregationExecutor——Hash Aggregation 模式：按 GROUP BY 键哈希分桶 → 桶内执行 SUM/COUNT/AVG/MAX/MIN 聚合函数 → Next emit 聚合结果。支持 HAVING 条件过滤和 COUNT DISTINCT 去重计数。
4. 实现 TopNExecutor——维护大小为 N 的堆，遍历 child executor 时只保留 Top N 个，等价 Sort+Limit 但性能更优（O(n log k) vs O(n log n)），与 Sort+Limit 进行性能对比验证。
5. 将 Sort/Aggregation/TopN/ExternalMergeSort 四种执行器接入 Planner 全链路，确保与 ORDER BY/GROUP BY/LIMIT 子句的正确对应关系。
6. 进行 Sort/Aggregation/TopN 性能基准测试（10 万行随机数据），输出排序/聚合模块设计文档（算法伪代码 + BufferPool 内存管理策略 + 复杂度分析）。
