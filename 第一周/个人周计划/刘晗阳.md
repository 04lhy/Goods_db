# 数据库系统小学期第一周周计划 — 刘晗阳（组长 / 索引层）

1. 阅读 BusTub 源码中 `storage/index/b_plus_tree.cpp` 及 `storage/page/b_plus_tree_*.cpp`，理解 B+ 树的页结构设计（基类、内部页、叶子页的键值布局）以及 BusTub 的索引模块整体架构，输出 B+ 树页结构类图。
2. 实现 B+Tree Page 基类、Internal Page（内部节点二分查找路由）和 Leaf Page（叶子节点 + 兄弟链表），完成页级别的单元测试，确保 `Lookup()` 能正确定位目标叶子页。
3. 实现 B+ 树的完整增删查操作——`Insert(key, rid)` 支持溢出分裂递归上溯、`Remove(key)` 支持半满合并或借位、`GetValue(key)` 点查、`RangeScan(start, end)` 区间扫描，以及 `IndexIterator` 封装叶子链表遍历，通过大量插入/删除/边界场景的单元测试。
4. 实现 Extendible Hash Index（全局深度 + 局部深度 + 目录页 + 桶页动态分裂），支持点查/插入/删除；对比 B+Tree 与 Hash 在点查、区间扫描、空间占用三个维度的性能差异，输出对比报告。
5. 结合新零售多仓储场景，设计索引选型方案（warehouse_id → Hash、order_date → B+Tree、sku → Hash、tracking_no → B+Tree），完成索引与 TableHeap 的联调回表，向 肖清峰交付稳定的索引接口（Insert / GetValue / RangeScan / Remove），并汇总全组进度、组织集成测试、编写阶段一总体文档。
