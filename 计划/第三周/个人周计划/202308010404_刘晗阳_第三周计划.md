# 数据库系统小学期第三周周计划 — 刘晗阳（组长 / Web 前端 + 后端完善）

1. 搭建 goods_db_studio Web 前端——单文件 HTML（index.html），零依赖，深色主题。布局：连接工具栏（主机/端口/用户/密码）→ 左侧对象树（数据库→表）→ 中央 SQL 编辑器 + 结果表格 → 底部状态栏。通过 REST API 与后端通信（/api/connect、/api/execute、/api/databases、/api/tables、/api/columns）。

2. 实现 HTTP API 服务层（goods_db_web）——基于 cpp-httplib（header-only），编译为独立可执行文件，无 Qt 依赖。路由：POST /api/connect（连接 goods_db 后端）、POST /api/disconnect、POST /api/execute（执行 SQL）、GET /api/databases、GET /api/tables、GET /api/columns、GET /（静态文件服务）。CORS 中间件支持跨域。手动 JSON 解析避免第三方依赖。

3. 实现后端协议客户端（GoodsDbProtocol）——Qt-free TCP 客户端，基于 POSIX socket。实现：Connect（非阻塞 + select 超时）、Authenticate（handshake + AUTH 包）、Execute（QUERY 包 → 读取响应 → ParseResponse）、Ping/Disconnect。响应解析：ParseResultSet（Columns/ColDef/Row/EOF 文本格式）、ParseOkError（OK/ERR 行）。

4. 修复 goods_db SQL 兼容性问题（P0 阻塞性）——Binder 新增 ResolveTableName() 辅助函数解析 db.table 格式；修复 SHOW COLUMNS 反引号处理和 FROM 偏移错误；修复 SHOW TABLES FROM 子句解析和数据库过滤。

5. 修复 goods_db SQL 兼容性问题（P1 功能性）——ExecutionEngine 新增 CREATE DATABASE / DROP DATABASE 支持（含 IF NOT EXISTS / IF EXISTS）；Catalog 新增多数据库架构（DatabaseInfo 层：Database → Table → Index），默认数据库名从 SQL 保留字 default 改为 goods_db；Executor 新增 affected_rows 传递（ExecutorContext → DML executor → ExecutionEngine → ConnectionHandler → SendOk）。

6. 修复运行时 Bug——UpdateExecutor/DeleteExecutor 死锁（两阶段执行：先收集 RID 再修改，避免迭代器读锁与 UpdateTuple 写锁冲突）；Optimizer PredicatePushdown 移除 Filter 节点导致 WHERE 子句失效（SeqScanExecutor 不支持 filter_predicate，暂禁用该优化规则）；ConnectionHandler 多语句支持（SplitSql 分号切分 + 逐条执行 + 单次 Flush）。

7. 组织全组联调——后端服务器（goods_db_server :3307）← 协议层（Protocol_text）← HTTP API（goods_db_web :9090）← 浏览器（index.html），完成连接→创建表→CRUD→WHERE 过滤→聚合→SHOW 命令→删除表的全链路端到端验证（15 项测试全部通过，97 个回归测试全部通过）。
