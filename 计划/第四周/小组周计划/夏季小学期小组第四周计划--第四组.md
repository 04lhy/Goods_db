- `drop database <name>` → 删除数据库
- `flush [hosts|logs|tables]` → 清除缓存/日志/表
- `reload` → 重载权限表（发送 `FLUSH PRIVILEGES`）
- `shutdown` → 关闭服务器（需确认）
（下午）实现 `goods_db_admin` 的服务器状态报告功能——通过网络协议向服务器请求统计信息（QPS/连接数/命中率等），如果服务器尚未暴露统计接口，则协调唐瑞涛在服务器端添加 `SHOW STATUS` SQL 命令。（下午）编写 goods_db_admin 完整命令测试（每种命令的输入输出验证）和与服务器的联调测试。

**李锦华负责：**
（上午）实现 `goods_db_dump` 全量备份工具——命令行程序：`goods_db_dump [options] [database [table...]]`。核心流程：
1. 连接服务器，获取 Catalog（所有数据库和表的列表）
2. 对每张选定的表：
   a. 发送 `SHOW CREATE TABLE table_name` → 获取完整 DDL（或通过 Catalog 构建 CREATE TABLE 语句）
   b. 发送 `SELECT * FROM table_name` → 获取全部数据行
   c. 生成 `INSERT INTO table_name VALUES (row1), (row2), ...` 批量 INSERT 语句
3. 写入输出文件（默认 stdout，`--result-file=path` 指定文件）
4. 选项：`--no-create-info`（不导出 CREATE TABLE）、`--no-data`（仅导出结构）、`--complete-insert`（INSERT 包含列名）、`--single-transaction`（在事务中导出保证一致性）
（下午）实现备份进度显示——大表备份时输出当前行数/总行数和进度百分比。实现恢复功能：`goods_db_dump` 生成的 `.sql` 文件可通过 `Execute(sql_file_content)` 进行恢复（利用第二阶段 SQL 解析器和执行器，或直接通过 goods_db shell 执行 SQL 文件）。（下午）编写全量备份/恢复的端到端测试——建表 → 插入 1 万行 → dump → DROP TABLE → 从 dump 文件恢复 → 验证数据完全一致。

**王宇航负责：**
