# 数据库系统小学期第二周周计划 — 王宇航（缓冲池替换策略 / Lock Manager / Transaction Manager）

1. 实现 LockManager——LockShared(txn, rid) / LockExclusive(txn, rid) / LockUpgrade(txn, rid) / Unlock(txn, rid) 核心接口，基于 2PL 协议（Growing 阶段只能加锁，Shrinking 阶段只能释放），支持表级意向锁（IS/IX/S/SIX/X），实现 waits-for graph + DFS 死锁检测与自动回滚。
2. 实现 TransactionManager——Begin() 创建事务分配 txn_id / Commit(txn) 写 WAL + 释放锁 / Abort(txn) 回滚 undo + 释放锁，实现 MVCC 可见性判断（基于时间戳的 tuple 创建/删除 txn_id 比较）。
3. 实现 HashJoinExecutor——左表构建哈希表（unordered_multimap<JoinKey, Tuple>），右表探测匹配，支持 INNER JOIN 和 LEFT OUTER JOIN。
4. 实现 NestedLoopJoinExecutor（NLJ）——左表全缓存 → 对每个左 tuple 重新遍历右表检查 JOIN 条件。实现 IndexJoinExecutor（与刘晗阳联调，外表 JOIN 键 → 内表 IndexScan）。
5. 进行三种 JOIN 执行器的性能对比测试（同数据量下 HashJoin vs NLJ vs IndexJoin 耗时对比）和结果一致性交叉验证。
6. 将 Lock Manager + Transaction Manager 集成到执行器——SELECT 根据隔离级别决定是否加 S 锁，INSERT/UPDATE/DELETE 加 X 锁，编写并发事务综合测试（Lost Update / Dirty Read / Non-Repeatable Read / Phantom Read 四种场景验证隔离级别正确性）。
