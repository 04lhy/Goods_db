# 4 拟实现的主要内容与要求

## 4.1 设计与实现任务 

设计与实现任务以项目组的形式组成，一个项目由 **4-6 个学生**组成。需选定一名项目组长，由组长确定该组的数据库设计工具、编程语言和开发工具。组内各成员必须统一使用选定的设计与实现环境。

### 4.1.1 设计一：设计与实现存储引擎
设计与实现 **hnusql** 的存储引擎，构建类似于 MySQL 的数据库服务器核心。存储引擎负责提供数据库访问接口的具体实现。

*   **参考实现：** MySQL 的相关代码大部分位于 `storage` 目录，集群和分区的处理类代码位于 `sql/` 目录。
*   **关键结构：** 必须实现 `handler` 类和 `handlerton` 结构。
*   **开发阶段：**
    1.  **生成引擎方法存根：** 创建可以被插入服务器的基本存储引擎框架。
    2.  **实现表管理操作：** 包括文件的创建（Create）、打开（Open）、关闭（Close）和删除（Delete）。
    3.  **实现数据读写操作：** 完成最基本的行级存储与读取。
    4.  **实现数据的更新和删除操作：** 完善基础的 DML 支持。
    5.  **添加索引功能：** 实现随机读写和区间查询能力。

### 4.1.2 设计二：存储引擎接口子系统
设计与实现存储引擎接口子系统，支持存储引擎的插件化扩展。

*   **结构组成：** 由 `handler` 类和 `handlerton` 结构组成，后者采用单态（Singleton）模型。
*   **核心定义：** 表处理类 `handler` 由 `sql/handler.h` 定义，并在 `sql/handler.cc` 中部分实现。
*   **设计目标：** 封装不同存储引擎的差异，为上层提供统一接口。

### 4.1.3 设计三：数据库管理子系统
设计与实现核心功能的管理工具，类似于 **`mysqladmin`**。

*   **核心功能：**
    *   创建/删除数据库及其所有表。
    *   显示服务器扩展状态消息、简短状态消息及版本信息。
    *   清除（Flush）主机缓存、日志及所有表。
    *   重载（Reload）授权表。
    *   管理线程：显示活跃线程列表、杀死指定线程。
    *   安全管理：修改管理员口令。
    *   服务维护：检查服务器运行状态、关闭服务器。

### 4.1.4 设计四：数据库备份子系统
设计与实现类似于 **`mysqldump`** 的备份工具，确保数据安全性。

*   **备份机制：** 采用 SQL 级别备份，将数据表导出为 SQL 脚本文件。
*   **功能要求：** 支持**全量备份**与**增量备份**。

### 4.1.5 设计五：日志子系统
设计与实现日志功能，记录服务器内部事件。

*   **日志类型：** 包括 **Error 日志**、**查询日志**及最广泛使用的**二进制日志（Binary Log）**。
*   **参考实现：** 参考 `sql/log.h` 中的 `MYSQL_LOG` 类。
*   **辅助工具：** 设计类似于 `mysqlbinlog` 的客户端程序，用于查看或解析二进制日志。

### 4.1.6 设计六：网络连接和网络通信协议子系统
设计与实现基于 **TCP/IP** 协议栈的网络通信层。

*   **功能职责：** 提供数据包的读写、解析和发送。
*   **参考实现：** 
    *   代码参考 `sql/protocol.cc`, `sql/protocol.h` 和 `sql/net_serv.cc`。
    *   **类层次结构：** `Protocol` 为基类，派生出 `Protocol_binary`（二进制）、`Protocol_local`（本地）和 `Protocol_text`（文本）子类。

### 4.1.7 设计七：线程和内存分配子系统
管理服务器启动过程及连接生命周期。

*   **核心流程：** 在 `mysqld.cc` 中的 `main()` 函数中实现自启动准备。
*   **功能职责：** 
    *   服务器启动后，线程子系统负责跟踪和分配线程。
    *   确保每个客户端连接都有对应的线程进行处理。

### 4.1.8 设计八：查询解析和查询优化
实现 RDBMS 的核心大脑，决定执行速度和效率。

*   **查询解析：** 从 `sql/sql_parse.cc` 中的 `mysql_parse()` 开始，初始化数据结构并调用 **`yyparse()`** 进行词法和语法分析。
*   **查询优化：** 从 `sql/sql_select.cc` 中的 `mysql_select()` 开始，目标是生成最优的查询执行计划（Query Plan）。

### 4.1.9 设计九：安全管理子系统
通过验证和访问控制保护服务器安全。

*   **身份验证：** 在 `sql/sql_parse.cc` 的 `check_connection()` 中实现用户连接请求的验证。
*   **访问控制：** 通过 `check_access()` 方法确认用户是否拥有对特定数据库、表或列的操作权限。
