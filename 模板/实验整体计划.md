# goods_db 实验整体计划

> **最后更新**：2026-07-13（基于实际代码与四周计划校准）

## 项目定位

基于 BusTub 技术架构，构建面向**新零售多仓储电商物流场景**的关系型数据库系统 **goods_db**，满足《数据库系统设计》课程 10 大设计模块要求。

- [x] Tuple / Schema / Value 类型系统 —— 8 种数据类型 + VARCHAR 变长字段 + 跨类型数值比较
- [x] TableHeap —— 表堆组织（双向链表多页容器，Insert/Update/Delete/SeqScan）

- [x] B+Tree Index —— 完整 CRUD + 区间扫描 + 页分裂合并
- [x] ExtendibleHashIndex —— 可扩展哈希索引（目录页 + 桶页，全局/局部深度）

- [x] `class goods_handler` —— 适配层（将各组件适配到 handler 虚函数）
- [x] 端到端集成测试 9/9 通过（FullEndToEnd 新零售仓库场景）

## 开发环境

```bash
# 1. 构建开发镜像
cd /home/lhy/桌面/Goods_db
docker compose build

# 2. 进入开发容器
docker compose run --rm bustub bash

# 3. 编译服务端
cd goods_db/build
cmake .. -DCMAKE_BUILD_TYPE=Debug
make -j$(nproc)

# 4. 运行测试
ctest --output-on-failure

# 5. 编译桌面客户端（需 Qt 6.2+）
cd goods_db_studio/build
cmake .. -DCMAKE_BUILD_TYPE=Debug
make -j$(nproc)
./goods_db_studio

# 6. 编译 Web 服务
make goods_db_web
./goods_db_web --port 9090

# 7. 启动服务器
cd goods_db/build
./src/sql/goods_db_server --port 3307 --datadir ./data
```
