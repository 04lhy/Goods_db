# 数据库系统小学期第四周周计划 — 王宇航（缓冲池替换策略 / 并发性能测试 + 系统调优）

1. 设计并编写全场景并发压力测试方案——5 种场景（只读点查 / 读写混合 8:2 / 纯写入 / 复杂查询 3 表 JOIN / 热点行竞争）× 3 种并发度（50/100/200）× 每组跑 60 秒，采集 QPS + 延迟分布（AVG/P50/P95/P99/MAX）+ 错误率（死锁回滚率）。
2. 编写压测客户端程序 tools/benchmark/stress_client.cpp——spawn N 个线程，每个独立连接服务器，按场景配置执行 SQL 操作，记录每次操作的耗时到 CSV（线程 ID/操作类型/开始时间/耗时 ms/是否成功）。编写结果汇总脚本 aggregate_results.py → 生成 Markdown 表格和 ASCII 图表。
3. 进行 Buffer Pool 命中率分析——pool_size 16/64/128/256/512 MB 五档配置 × 50/100/200 三种并发度，采集命中率×吞吐量，找到性价比拐点。进行索引选型验证——在 50 万行新零售数据上对比 B+Tree vs Hash 在真实业务查询中的表现。
4. 输出并发性能测试最终报告——QPS-并发数特性曲线、延迟分布百分位图、Buffer Pool 命中率-吞吐量相关性分析、锁争用热力图、性能瓶颈诊断（当前系统的主要瓶颈/优化优先级/预期改进效果）。
5. 编写系统调优指南——针对 OLTP 高并发写入 / OLAP 分析查询 / 均衡混合负载三种典型业务场景的推荐配置（Replacer 类型/Buffer Pool 大小/隔离级别/日志策略/网络参数）。
6. 协助全组进行最终集成测试（负责并发事务场景压测 Demo 和数据展示），参与全组最终验收演示（并发 Demo + 性能报告讲解）。
