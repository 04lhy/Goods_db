# 数据库系统小学期第二周周计划 — 唐瑞涛（项目骨架 / 工具链 / CI）

1. 集成 libpg_query 到 goods_db 工程，封装 `Parser::Parse(sql)` → AST 树接口，定义 AST 节点类型枚举和 AstVisitor 遍历访问器基类，编写 AST 打印工具（AstPrinter）方便调试，交付全组作为查询引擎的前置依赖。
2. 实现 Expression 表达式体系——二元运算（算术/逻辑/比较）、一元运算（NOT/负号）、常量值、列引用、函数调用，提供 `Expression::Evaluate(Tuple*, Schema*)` → Value 运行时计算，编写全量表达式单元测试。
3. 实现 EXPLAIN 和 EXPLAIN ANALYZE 命令——输出优化前后计划树的缩进文本表示和执行器实际耗时/emit 行数统计。
4. 实现 goods_db 命令行交互工具（goods_db_shell）——基于 linenoise 提供 readline 风格 SQL 输入（历史/补全），集成 Execute(sql) 统一入口，格式化输出结果集表格。
5. 更新 CI 流水线——新增查询引擎层 goods_db_sql 库的编译和测试目标，确保 push 触发全量编译+测试+format/lint 检查，辅助全组解决跨模块链接问题。
6. 编写 Benchmark 工具宏（BENCHMARK_START/END → 微秒级耗时），供索引和执行器性能对比使用。输出第二阶段 SQL 语法支持文档（完整 BNF 描述 + 与 MySQL 差异对照表）。
