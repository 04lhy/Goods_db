# 数据库系统小学期第三周周计划 — 王宇航（缓冲池替换策略 / Lock Manager / 并发控制）

1. 完善 Lock Manager 以支持网络层多线程场景——扩展表级意向锁（IS/IX/S/SIX/X）用于 DDL/DML 互斥，实现锁等待超时机制（lock_wait_timeout 可配置，超时自动回滚，作为死锁检测的补充），暴露统计接口（GetLockCount / GetDeadlockCount / GetAvgWaitTime）供管理面板展示。
2. 完善 Transaction Manager——支持 SAVEPOINT 保存点（事务内部分回滚）和 RELEASE SAVEPOINT，实现 XID 分配器（单调递增 64 位 ID + 持久化到 header page），为 Binary Log 提供事务 ID。
3. 将 Lock Manager 集成到 ConnectionHandler——每个连接维护活跃事务，SELECT 根据隔离级别加 S 锁，INSERT/UPDATE/DELETE 加 X 锁 + 表级 IX 锁，DDL 加表级 X 锁。实现 START TRANSACTION / COMMIT / ROLLBACK / SET TRANSACTION ISOLATION LEVEL SQL 命令的 Parser → Executor 全链路。
4. 实现 SERIALIZABLE 隔离级别——所有 SELECT 自动加 S 锁 + 范围锁（gap lock）阻止 Phantom Read，Lock Manager 相应扩展 gap lock 支持。
5. 进行全场景并发压力测试——50/100/200 三种并发度 × 5 种场景（只读/读写混合/纯写入/复杂查询/热点竞争），采集 QPS/延迟分布(P50/P95/P99)/死锁率/锁等待时间。
6. 协助刘晗阳进行桌面端网络联调——确保带事务的查询（BEGIN → SELECT → UPDATE → COMMIT）通过桌面端 SQL 编辑器执行时行为正确。输出 Lock Manager + Transaction Manager 完整设计文档（2PL 状态机 + 死锁检测算法伪代码 + MVCC 可见性决策表 + 四种隔离级别实现细节对比）。
