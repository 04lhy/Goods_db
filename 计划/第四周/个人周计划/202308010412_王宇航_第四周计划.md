# 数据库系统小学期第四周周计划 — 王宇航（并发性能测试 + 系统调优）

1. 设计并编写全场景并发压力测试方案——5 种场景（只读点查 / 读写混合 8:2 / 纯写入 / 复杂查询 3 表 JOIN / 热点行竞争）× 3 种并发度（50/100/200）× 每组跑 60 秒，基于物流电商数据模型，采集 QPS + 延迟分布（AVG/P50/P95/P99/MAX）+ 错误率（死锁回滚率）。

2. 编写压测客户端程序 tools/benchmark/stress_client.cpp——spawn N 个线程，每个独立连接服务器，按场景配置执行 SQL 操作，记录每次操作的耗时到 CSV（线程 ID/操作类型/耗时 ms/是否成功）。编写结果汇总脚本生成 Markdown 表格。

3. 进行 Buffer Pool 命中率分析——pool_size 16/64/128/256 MB 四档配置 × 50/100/200 三种并发度，采集命中率×吞吐量，找到性价比拐点。输出"命中率 vs pool_size vs 并发数"三维对比表。

4. 进行物流电商场景索引选型验证——在 50 万行数据上，对比 B+Tree vs Hash 索引在真实业务查询中的表现：库存点查（Hash 胜）、仓库汇总 GROUP BY（Hash 适合 GROUP BY 键）、订单日期范围查询（B+Tree 必须）、物流运单号查询（B+Tree 适合唯一键排序）。输出索引选型建议。

5. 输出并发性能测试最终报告——含 QPS-并发数特性曲线、延迟分布百分位图（P50/P95/P99）、Buffer Pool 命中率-吞吐量相关性分析、死锁率随并发度变化趋势、性能瓶颈诊断总结。编写系统调优指南——针对 OLTP 高并发写入 / OLAP 分析查询 / 均衡混合负载三种场景的推荐配置。
