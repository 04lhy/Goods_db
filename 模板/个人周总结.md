### 学习笔记：
1. **主要目录介绍：**
   - **a. BUILD目录：** 编译和安装脚本目录，里面有各种平台使用的编译脚本。
   - **b. Client目录：** 包括 `mysql.cc`, `mysqlcheck.c`, `mysqladmin.cc`, `mysqlshow.c` 等文件，是客户端程序所在的目录。
   - **c. Storage目录：** MySQL 各类存储引擎 (storage engine) 代码。MySQL 实现了一个抽象接口层，叫做 `handler` (sql/handler.h)，其中定义了接口函数，比如：`ha_open`, `ha_index_end`, `ha_create` 等，存储引擎需要实现这些接口才能被系统使用。现在比较常用的是 `innodb`，最早的是 `myisam`。
   - **d. Mysys目录：** 包含了对于系统调用的封装，其作用是能够容易实现平台间的跨越。
   - **e. Sql目录：** 包含了数据库的主程序 `mysqld`，是源码中需要经常变化的目录之一。这里对各类 SQL 语句进行解析和实现。其中 `sql_lex.cc` 是词法解析模块，`sql_yacc.yy` 是语法解析模块。
   - **f. vio目录：** 封装了 virtual I/O 接口，主要是各种协议的网络操作。

2. **从 main.cc 出发，逐步深入分析：**
   入口函数 `main()` 直接调用了 `mysqld_main()`。

```cpp
/*
  main() for mysqld.
  Calls mysqld_main() entry point exported by sql library.
*/
extern int mysqld_main(int argc, char **argv);

int main(int argc, char **argv)
{
    return mysqld_main(argc, argv);
}
```

Server 启动的代码都在这里，直到最后的 `handle_connections_sockets()`。在 `mysqld_main` 中，会处理实例句柄、初始化系统字符集（`system_charset_info`）、调用 `my_init()` 并在 NT 系统下处理服务初始化逻辑。

### 学习笔记：
**MySQL 架构：**
- **Client 层：** 处理连接、协议。
- **MySQL Server 层：** 包括 Connection/Thread Pool、Sql Interface、Parse、Optimizer、Cache。
- **Storage Engines 层（可插拔）：** 如 InnoDB、InMemory、MyISAM 等。

MySQL 开启一个线程对 3306 端口进行监控，等待客户端请求触发 `add_connection` 回调。

```cpp
void handle_connections_sockets()
{
    // 监听连接
    new_sock = mysql_socket_accept(key_socket_client_connection, sock, (struct sockaddr *)(&cAddr), &length);
    if (mysql_socket_getfd(sock) == mysql_socket_getfd(unix_sock))
        thd->security_ctx->set_host((char*) my_localhost);
    
    // 创建连接
    create_new_thread(thd);
}

static void create_new_thread(THD *thd) {
    thd->thread_id = thd->variables.pseudo_thread_id = thread_id++;
    // 线程进入线程调度器
    MYSQL_CALLBACK(thread_scheduler, add_connection, (thd));
}
```
