# 数据库系统小学期第二周周计划 — 肖清峰（引擎接口层 / 集成联调）

1. 定义逻辑计划节点全部 14 种 PlanNode 派生类——SeqScan / IndexScan / Filter / Projection / Insert / Update / Delete / HashJoin / NLJ / IndexJoin / Aggregation / Sort / Limit / Values / Window，确定每种节点的字段（output_schema + children + 特性字段）和构造方式。
2. 实现 Planner——`Planner::Plan(BoundStatement*)` → `AbstractPlanNode*`，从 BoundStatement 生成初始逻辑计划树（未优化的直接翻译），覆盖全部 BoundStatement 类型到对应 PlanNode 的转换规则。
3. 实现 DML 执行器——InsertExecutor（TableHeap::InsertTuple + 索引同步更新）、UpdateExecutor（TableHeap::UpdateTuple + 索引同步更新）、DeleteExecutor（TableHeap::MarkDelete + 索引标记删除），编写 DML 全链路单元测试。
4. 实现 FilterExecutor / ProjectionExecutor / LimitExecutor / SeqScanExecutor 四种基础执行器，确保 Volcano 模型 Init/Next 接口实现正确。
5. 实现 Planner 全链路调度框架——`Planner::CreatePlan(PlanNode)` → `unique_ptr<AbstractExecutor>`，后序遍历 PlanNode 树创建执行器对象并串联 child executor。
6. 打通"SQL 文本 → Parser → Binder → Planner → Optimizer → Executor → 结果集"端到端全链路，编写覆盖 SELECT/INSERT/UPDATE/DELETE/JOIN/聚合/排序/子查询的完整集成测试。输出 Planner + 执行器设计文档（PlanNode 类层次图 + Volcano 生命周期 + 依赖关系图）。
