## 已修复的问题

### 1. ✅ TablePage InsertTuple 布局 bug（已修复）

**文件**: `src/storage/page/table_page.cpp`

**根因**: slotted page 布局实现不正确。原代码将 tuple 放在 slot array 之后（两者都从 header 向高地址增长），导致第一个 tuple 插入后与 slot entry 重叠。

**修复方案**: 重写为标准 slotted page 布局：
- Slots 从 header 向高地址增长
- Tuples 从 PAGE_SIZE 向低地址增长
- 插入新 tuple：`new_offset = lowest_tuple - size`（向 PAGE_SIZE 方向增长）
- 空闲空间 = `lowest_tuple - slot_array_end`

**验证**: page_test 7/7 pass, table_test 2/2 pass

### 2. ✅ Tuple::GetValue VARCHAR 反序列化 heap-buffer-overflow（已修复）

**文件**: `src/storage/table/tuple.cpp`

**根因**: `CreateFromValues` 中 `var_offset_ptr` 初始化位置错误，使用了已被累加修改的 `var_offset_offset`（指向固定区末尾），而非固定区起始地址。导致 VARCHAR offset 写入到错误位置，GetValue 读取到垃圾数据。

**修复方案**: 明确分离固定数据和变长偏移区域：
- 布局: `[Header][NullBitmap][FixedData][VarOffsets][VarData:[len][data]...]`
- Var offset 区域起始 = `fixed_offset + 所有非VARCHAR列的固定大小之和`
- 写入和读取逻辑统一使用此布局

**验证**: type_test 7/7 pass (原 4/5), table_test 的 InsertAndGet 通过

### 3. ✅ ClockReplacer 单元测试（已修复）

**文件**: `test/buffer/buffer_test.cpp`

**说明**: Clock 算法在 Unpin 时设置 ref_bit=true，第一次 Victim 调用只清除 ref_bits 而不驱逐。测试已改为循环调用 Victim 直到找到 victim。buffer_test 6/6 pass。

### 4. ✅ TableHeap InsertTuple 断言失败（随问题1修复）

**文件**: `src/storage/table/table_heap.cpp:84`

**说明**: 依赖 TablePage::InsertTuple 成功，TablePage 的 bug (问题1) 修复后自动解决。

### 5. ✅ LRU-K Replacer 未使用字段警告（已修复）

**文件**: `src/include/buffer/lru_k_replacer.h:49`

**修复**: 移除未使用的 `num_frames_` 字段。编译零警告。

### 6. ✅ page_test 栈溢出 SegFault（已修复）

**文件**: `test/storage/page/page_test.cpp`

**根因**: 测试中 `char page_data[PAGE_SIZE]` (16KB) 栈分配 + ASan redzone 导致栈溢出。

**修复**: 改用 `std::make_unique<char[]>(PAGE_SIZE)` 堆分配。
