# 数据库系统小学期第二周周计划 — 刘晗阳（组长 / 索引层 + 桌面客户端）

1. 实现 IndexScanExecutor——支持 B+Tree（等值 + 范围扫描）和 HashIndex（仅等值点查）两种索引的查询执行器，Init 时根据查询条件选择合适索引定位起始页，Next 时沿索引链表遍历并通过 TableHeap::GetTuple(rid) 回表获取完整 tuple，编写完整单元测试覆盖等值/范围/回表验证场景。
2. 实现 IndexJoinExecutor——外表每行 tuple 的 JOIN 键值作为内表索引查找 key，内表通过 IndexScan 定位匹配行完成 JOIN 操作，与 NLJ/HashJoin 结果交叉验证确保一致性。
3. 实现 Optimizer 索引相关优化规则——SeqScan → IndexScan 改写（WHERE 包含索引列的等值/范围条件时触发）、NLJ → IndexJoin 改写（内表 JOIN 列有索引时触发）、NLJ → HashJoin 改写（等值 JOIN 无索引时触发），每条规则编写独立单元测试验证改写前后的计划树和执行结果一致性。
4. 实现 WindowFunctionExecutor——ROW_NUMBER() / RANK() / DENSE_RANK() 三种排名窗口函数，PARTITION BY 分窗 + ORDER BY 窗内排序，以及 SUM() OVER (ROWS BETWEEN ... AND ...) 窗口聚合（累积和/滑动窗口），编写分组/排序/帧规范的综合测试。
5. 协同肖清峰完成 Optimizer 剩余规则集成（谓词下推 + 列裁剪 + Sort+Limit→TopN），确保 EXPLAIN 命令可验证优化规则正确应用。
6. 进行 B+Tree vs Hash 索引性能对比测试（点查 / 区间扫描 / 空间占用三维度），结合 50 万行新零售场景数据设计索引选型方案（每张表的推荐索引类型和理由），输出对比报告。
7. 汇总全组进度和交付物，编写阶段二总体设计文档（查询引擎全景架构图 + Expression/BoundStatement/PlanNode/Executor 四大类层次图 + 数据流图 + 新零售 SQL 扩展语法文档），组织周五端到端集成测试验收。
