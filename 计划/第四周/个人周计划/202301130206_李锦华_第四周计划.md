# 数据库系统小学期第四周周计划 — 李锦华（备份子系统 + 日志完善）

1. 实现 goods_db_dump 全量备份工具——遍历 Catalog 获取所有数据库和表的列表 → 对每张表获取 DDL（CREATE TABLE 语句）→ SELECT * 获取全部数据 → 生成批量 INSERT 语句（每 100 行一批）→ 写入输出文件。支持 --no-create-info / --no-data / --complete-insert / --result-file 选项。

2. 实现增量备份功能——基于 Binary Log 位点：记录上次备份的 binlog 文件名和 position → 下次备份从上次位点读取 binlog 事件 → 将 DML 事件转换为 SQL 语句输出。支持 --incremental --since-position 选项。

3. 实现恢复功能——读取 .sql 备份文件 → 分条执行 SQL 进行恢复。编写全量备份→恢复→增量备份→恢复的全链路闭环测试，模拟生产环境灾难恢复流程（建库→写数据→全量备份→增量备份→模拟故障→全量恢复→增量恢复→数据一致性校验）。

4. 编写备份恢复操作手册——全量/增量/恢复的操作流程、备份策略配置建议（cron 定时示例、备份文件轮转策略、监控告警建议）。

5. 日志子系统完善——确保 ErrorLog / QueryLog / BinaryLog 在全系统集成测试中正确记录，24 小时稳定性验证（无内存泄漏、文件轮转正常、索引文件与 binlog 文件一致）。输出备份子系统完整设计文档。
