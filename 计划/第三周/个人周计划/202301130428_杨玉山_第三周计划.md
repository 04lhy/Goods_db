# 数据库系统小学期第三周周计划 — 杨玉山（安全管理子系统）

1. 设计 goods_db 安全模型——goods_db.user 用户账户表（host/user/password_hash/salt，SHA-256 密码哈希 + 随机盐值）、goods_db.db 数据库级权限表（host/user/db_name + SELECT/INSERT/UPDATE/DELETE/CREATE/DROP/INDEX/GRANT 8 种权限）、goods_db.tables_priv 表级权限表（host/user/db_name/table_name + 列级权限），ACL 系统表以普通 TableHeap 存储但标记为系统表（不可被普通 DROP TABLE 删除）。

2. 实现 AuthManager 用户认证——check_connection(host, user, password) → bool：查 goods_db.user 验证 host+user 存在 → SHA-256(password+salt) 比对 password_hash → 返回认证结果。启动时自动创建默认 root 用户（host=localhost, user=root, password 空）。支持认证失败记录（IP + 用户名 + 失败原因写入 ErrorLog）。

3. 实现 GRANT / REVOKE SQL 命令——扩展 Parser（添加 GRANT/REVOKE 关键字）→ Binder（BoundGrantStatement / BoundRevokeStatement）→ Planner → Executor 全链路。GRANT SELECT,INSERT ON db.table TO 'user'@'host' → 更新权限表；REVOKE INSERT ON db.table FROM 'user'@'host' → 撤销权限。

4. 实现用户管理 SQL 命令——CREATE USER / DROP USER / ALTER USER / SET PASSWORD，以及 FLUSH PRIVILEGES（重新从系统表加载权限缓存到内存 HashMap 提升检查性能）。

5. 将 AuthManager 接入 ConnectionHandler 认证流程——客户端连接后服务端发送 greeting 包，客户端返回 AUTH 包（user\0password\0db\0），服务端调用 AuthManager::CheckConnection 验证，失败返回 ERR 并关闭连接。

6. 编写安全管理端到端测试——创建用户 → 用新用户连接成功 → 授予 SELECT 权限 → 查询成功 → 尝试 INSERT 被拒 → 撤销权限 → 查询被拒 → 删除用户 → 连接失败。输出安全管理子系统完整设计文档（ACL ER 图 + 认证流程图 + 权限检查矩阵 + 密码安全方案）。
