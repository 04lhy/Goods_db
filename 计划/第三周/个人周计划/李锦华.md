# 数据库系统小学期第三周周计划 — 李锦华（磁盘与缓冲层 / 日志子系统 + 桌面客户端辅助）

1. 设计日志系统三层架构——ErrorLog（服务器错误/警告/信息，格式 [timestamp][level][thread_id] message，可配置级别过滤和输出目标）、QueryLog（所有 SQL 查询记录，格式 [timestamp][thread_id][user@host][db][exec_time][rows] sql_text，支持按用户/数据库/最小耗时过滤）、BinaryLog（WAL 消费端，QUERY/ROW_INSERT/ROW_UPDATE/ROW_DELETE/XID 五种事件类型，事务提交前刷盘 + fsync，文件轮转 1GB 切割）。
2. 实现 ErrorLog 和 QueryLog 基础功能——格式化输出 + 时间戳 + 线程 ID → 写入文件 + stderr，文件轮转（100MB 切割），实现日志管理 SQL 命令（SHOW LOGS / SHOW BINLOG EVENTS / PURGE BINARY LOGS / FLUSH LOGS / RESET MASTER）。
3. 实现 BinaryLog 核心——BinlogWriter 类：WriteEvent(event_type, txn_id, payload) → 构建事件包（时间戳 + server_id + 事件类型 + txn_id + payload + CRC32 校验和）→ 写入当前 binlog 文件 → 超 max_binlog_size 时 Rotate 切换到序号文件，实现 binlog 索引文件（记录所有 binlog 文件名和起始位点）。
4. 实现 goods_db_binlog 命令行工具——goods_db_binlog [options] binlog_file，支持 --start-position / --stop-position / --verbose / --base64-output 选项，参照 mysqlbinlog 格式输出（每事件一行注释 + QUERY 事件原始 SQL + ROW 事件解码）。
5. 将日志系统接入全服务器模块——ErrorLog 自动记录启动/关闭/错误/异常，QueryLog 自动记录每条 SQL 执行，BinaryLog 在事务提交前写入 binlog 事件。
6. 协助刘晗阳进行桌面客户端网络对接——确保 LogViewer 通过服务器端 SHOW LOGS/SHOW BINLOG EVENTS 接口获取日志数据并在桌面端正确展示。输出日志子系统完整设计文档（架构图 + 事件格式字节级规范 + 管理命令参考）。
