# goods_db 小组第二周计划

### 总目标：

1. 基于第一阶段完成的存储引擎与接口层，完成 goods_db 查询引擎层（设计八）的全部编码与测试。
2. 集成 libpg_query 实现完整 SQL 解析链路（词法分析 → 语法分析 → AST → BoundStatement）。
3. 实现 Binder（语义绑定）、Planner（逻辑计划生成）、Optimizer（查询优化器）三层完整链路。
4. 实现全部 13 种 Volcano 模型执行器，覆盖全表扫描、索引扫描、DML、JOIN、聚合、排序、窗口函数等操作。
5. 扩展新零售场景 SQL 支持（仓库 JOIN、库存汇总、物流状态追踪、销售趋势分析），周五完成端到端查询集成测试并通过验收。

**李锦华负责：**

**王宇航负责：**
