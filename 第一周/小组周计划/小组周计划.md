# goods_db 小组第一周计划

### 总目标：

1. 基于 BusTub 技术架构，完成 goods_db 存储引擎核心层（设计一）与引擎接口层（设计二）的全部编码与测试。
2. 搭建统一的 C++17 + Clang 15 + CMake 工程骨架，配置 Docker 开发环境与 CI 流水线，确保全组在一致环境下协作。
3. 实现 DiskManager → BufferPoolManager → TableHeap → B+Tree/HashIndex 的完整存储链路，支持行级 CRUD 与索引加速。
4. 参照 MySQL handler/handlerton 模式，定义并实现引擎接口抽象层，支持引擎插件化注册与切换。
5. 周五完成端到端集成测试（建表 → 插入 10000 行 → 索引查询 → 全表扫描验证 → 删除表）并通过验收。
