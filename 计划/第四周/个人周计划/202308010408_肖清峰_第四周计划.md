# 数据库系统小学期第四周周计划 — 肖清峰（集成测试 + 验收演示）

1. 编写全系统端到端集成测试——覆盖 10 大场景：服务器启动→客户端连接→用户认证→DDL 建表（物流电商 7 张表）→DML 数据操作（INSERT/UPDATE/DELETE）→复杂查询（JOIN/聚合/排序）→事务（BEGIN→COMMIT/ROLLBACK）→权限（GRANT→操作→REVOKE）→日志完整性验证→服务器关闭重启数据持久化。每个场景输出 PASS/FAIL 和详细日志。

2. 编写自动化集成测试脚本 test/integration/full_system_test.sh——Shell 脚本自动执行全部测试场景，覆盖边界情况（空表操作/超长字符串/极端数值/并发一致性/崩溃恢复），输出测试报告。

3. 编写 goods_db_admin 命令行管理工具——基于 GoodsDbProtocol 客户端实现独立 CLI 程序，覆盖核心管理命令：ping / status / version / processlist / create database / drop database / flush / shutdown。支持 --host/--port/--user/--password 连接参数。

4. 组织全系统最终验收演示——完整操作流程：编译启动 → 物流电商 DDL 建表 → 数据加载 → 7 大业务查询 → 事务并发 → 权限管理 → 备份恢复 → 日志展示 → Web 前端全功能演示。输出集成测试验收报告。

5. 记录验收中的问题和反馈，整理为改进清单（P0 必须修复 / P1 下版本优化）。
