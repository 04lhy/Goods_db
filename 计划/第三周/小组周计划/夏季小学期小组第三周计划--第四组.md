**肖清峰负责：**
（上午）研读 MySQL 8.0 `sql/protocol.h` 和 `sql/protocol.cc` 源码，深入理解：（1）`Protocol` 抽象基类的虚函数表——`store(T)` 存储结果集数据到发送缓冲区、`store_result()` 发送结果集元数据（列数、列名、列类型）、`send_ok()` 发送成功响应（affected_rows + last_insert_id + server_status + warning_count）、`send_error()` 发送错误响应（error_code + sql_state + error_message）、`send_eof()` 发送结果集结束标记。（2）`Protocol_text` 文本协议——数据按文本行发送（列值用 `\t` 分隔、行用 `\n` 分隔），客户端按行解析。（3）`Protocol_binary` 二进制协议——数据按紧凑二进制格式发送（固定长度类型直接写入、变长类型先写长度再写数据），效率更高。（下午）设计 goods_db 网络协议包格式：
```
Packet Header (4 bytes):
  - payload_length (3 bytes): 有效载荷长度（最大 16MB）
  - sequence_id (1 byte): 包序号（递增，用于请求-响应匹配）

Command Packet (client → server):
  Header + command_byte (1 byte: 0x03=QUERY / 0x01=QUIT / 0x0E=PING) + sql_string

Response Packet (server → client):
  - OK Packet: Header + 0x00 + affected_rows (varint) + last_insert_id (varint) + status_flags (2 bytes) + warnings (2 bytes) + info (string)
  - ERR Packet: Header + 0xFF + error_code (2 bytes) + sql_state_marker '#' + sql_state (5 bytes) + error_message (string)
  - ResultSet: ColumnCount Packet + N × ColumnDefinition Packet + EOF + N × RowData Packet + EOF
```
（下午）编写协议包格式定义文档（`docs/design/network_protocol.md`），包含每种包的字节级布局和示例。与刘晗阳对齐网络客户端库（`goods_db_client`）的接口签名。
