# goods_db 小组第一周周报

### 总目标：
1. 基于 BusTub 技术架构，完成 goods_db 存储引擎核心层（设计一）与引擎接口层（设计二）的全部编码与测试。

**学习总结：**

第二天各模块进入深度编码阶段。

**唐瑞涛**交付了通用组件库 `src/common/`：(1) `RWLatch` 读写锁（`RLock/RUnlock/WLock/WUnlock`，基于 `std::shared_mutex`，用于 BufferPool 的 Page 级并发控制）；(2) `Config` 配置管理类（从 `goods_db.conf` 解析 buffer_pool_size/page_size/data_directory/port/log_level 等参数）；(3) 日志宏 `LOG_INFO/LOG_WARN/LOG_ERROR`（基于 fmt 库，含时间戳和日志级别前缀）及断言宏 `GOODS_DB_ASSERT`；(4) 编译期常量 `config.h`：`PAGE_SIZE`(16384)、`BUFFER_POOL_DEFAULT_SIZE`(64MB)、`MAX_TUPLE_SIZE`(8192)；(5) 通用工具 `HashUtil`（murmur3 封装）和 `StringUtil`（split/trim）。所有组件配备单元测试。

**李锦华**上午完成 DiskManager 多文件支持收尾与单元测试验证，下午启动 BufferPoolManager 关键路径开发：定义 `Page` 类（`page_id_` + `pin_count_` + `is_dirty_` + `data_[PAGE_SIZE]` + `RWlatch`），实现 `FetchPage(page_id)`（页表查找 → 命中则 pin_count++；未命中则从 DiskManager 读入空闲帧，无空闲时调用 Replacer 淘汰）、`UnpinPage(page_id, is_dirty)`（pin_count--，归零标记可淘汰）、`NewPage()`（AllocatePage + 载入空闲帧）、`DeletePage` / `FlushPage` / `FlushAllPages`。建立页表 `unordered_map<page_id, frame_id>` + 互斥锁保护，维护 `free_list_` 空闲帧列表。

**杨玉山**上午完成 TablePage 收尾，下午实现 Tuple 行格式与类型系统：(1) `TypeId` 枚举——`INTEGER/BIGINT/SMALLINT/TINYINT/BOOLEAN/DECIMAL/TIMESTAMP/VARCHAR`；(2) `Value` 类——`union { int32_t i32; int64_t i64; double f64; bool b; char str[256]; }` + `TypeId tag` + 比较运算符 + `ToString()` + `SerializeTo`；(3) `Column` 定义——`column_name + TypeId + max_length + IsNullable`；(4) `Schema` 定义——`vector<Column>` + `GetColIdx` + `GetColumnCount`；(5) `Tuple` 序列化格式——固定列在前（直接值），变长列在后（offset+len 对），重点支持 VARCHAR 变长字段以适配新零售场景（仓库名、地址、运单号）。

**刘晗阳（组长）**在页结构基础上实现了 B+ 树的完整增删查操作：(1) `Insert(key, rid)`——从根页 `Lookup(key)` 路由到目标叶子页 → 插入 KV → 若溢出创建新叶并均分 → 新叶首 key 上溯插入父节点 → 递归分裂到根；(2) `Remove(key)`——定位叶子页 → 删除 KV → underflow 检测 → `AdjustRoot` 根收缩；(3) `GetValue(key)`——根页出发逐层二分查找定位叶子页 → 二分查找返回 RID；(4) `RangeScan(start, end)`——`Lookup(start)` 定位起始叶 → 沿 `next_page_id_` 链表遍历收集结果；(5) `CreateNewRoot`——树高增长的唯一途径，保证所有叶子在同一层；(6) `FindLeaf`——从根到叶的逐层路由，正确管理 Fetch/Unpin 配对。

**肖清峰**上午确认与全组的接口签名并定稿引擎接口设计文档，下午实现 `sql/handlerton.h` 和 `sql/handlerton.cc`：(1) `struct handlerton`——`name + init/deinit 生命周期回调 + create_handler 工厂指针 + commit/rollback 事务回调预留`；(2) 全局引擎注册表 `unordered_map<string, handlerton*>`（线程安全）；(3) `register_engine/get_engine/get_all_engines`。完成注册/查找/工厂调用的单元测试。

**王宇航**上午定义了 `class Replacer` 虚基类（`Victim/Pin/Unpin/Size/Name`）并实现了 `ClockReplacer`（环形缓冲 + 引用位指针扫描：`Victim()` 遇到 ref_bit=0 淘汰，ref_bit=1 清零跳到下一帧），下午实现了 `LRUKReplacer`（追踪每帧最近 K 次访问时间戳，驱逐 backward k-distance 最大的帧），并将 Replacer 虚基类头文件交付李锦华。

## 存在的问题与难点

1. **集成测试 BPlusTreeIndex 用例**（rnd_pos 返回 false）：B+Tree GetValue 返回的 RID 与 TableHeap 实际 RID 不匹配，索引构建时的 RID 与存储时的 RID 一致性问题，已定位，刘晗阳/肖清峰/杨玉山约定第二周初集中解决。
2. **集成测试 UpdateAndDelete 用例**：TableHeap 遍历时可能超时，已标记预存问题。
3. **B+Tree 删除的借位与合并未完整实现**：当前 `HandleUnderflow` 仅调用 `AdjustRoot` 收缩根，未实现 rotate 和 merge。极端删除场景下空间利用率可能偏低，刘晗阳已列入第二周优化计划。
4. **Hash Index 的桶合并与目录收缩未实现**：删除后目录可能膨胀，长期运行不回收空间。已列入第二周优化计划。
5. **索引定量性能对比未完成**：B+Tree vs Hash 在点查/区间/空间占用的多档数据量定量对比测试纳入第二周。
